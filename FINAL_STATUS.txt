     • Queue abstraction supporting RQ and Celery

     • Redis persistence with configurable backend
     • Global singleton pattern for queue access
     
  🧠 backend/services/highlight_detector.py (350+ lines)
     • 3-part ML scoring algorithm
     • Audio analysis (RMS + spike detection)
     • Motion detection (frame-to-frame analysis)
     • Text analysis (multi-language keyword matching)
     • Scoring formula: 0.5×audio + 0.2×motion + 0.3×text
     • Language support: English, French, Spanish, German
     
  ⚙️ backend/queue/worker.py (240+ lines)
     • Main task processor for video analysis
     • Chunked processing (30-minute segments)
     • Parallel worker support
     • Real-time progress tracking (5% granularity)
     • Complete pipeline: Download → Segment → Process → Filter → Generate
     
  🔌 backend/api/advanced_routes.py (140+ lines)
     • 3 REST API endpoints
     • POST /api/generate/twitch/advanced (start processing)
     • GET /api/status/twitch/{job_id} (check progress)
     • DELETE /api/jobs/{job_id} (cancel job)
     • Complete validation and error handling

🚀 QUICK START (10 MINUTES)
═════════════════════════════════════════════════════════════════════════════

STEP 1: Start Services (3 minutes)
  
  # macOS/Linux:
  chmod +x start_services.sh
  ./start_services.sh rq
  
  # Or with Docker:
  docker-compose up -d

STEP 2: Run Workers (2 minutes)
  
  # Terminal 1: Worker
  rq worker
  
  # Terminal 2: API
  cd backend
  uvicorn main:app --reload

STEP 3: Test API (3 minutes)
  
  # Terminal 3: Test
  curl -X POST http://localhost:8000/api/generate/twitch/advanced \
    -H "Content-Type: application/json" \
    -d '{
      "url": "https://www.twitch.tv/videos/123456789",
      "max_clips": 5,
      "language": "en"
    }'

STEP 4: Check Progress (2 minutes)
  
  curl http://localhost:8000/api/status/twitch/{job_id}

PHASE 3: Update Configuration (2 minutes)
  
  Create/update backend/.env with:
    QUEUE_BACKEND=rq

  rq worker &
  uvicorn backend.main:app --reload

  Use curl commands or Postman
